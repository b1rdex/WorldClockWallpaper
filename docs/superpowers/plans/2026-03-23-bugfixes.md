### Task 1: Fix day/night terminator formula in map.html

**Files:**
- Modify: `WorldClockWallpaper/Resources/map.html` lines 30, 58–78

The bug: `SunCalc.getPosition(date, 0, subsolarLon).altitude` at the subsolar meridian/equator
does NOT equal the sun's declination. It equals `π/2 − |declination|`, which produces the wrong
antisolar latitude, flipping day/night near the poles vs equator.

The fix: compute declination directly from day-of-year using the standard approximation.

- [ ] **Step 1: Replace `getAntisolarPoint` in map.html**

  In `WorldClockWallpaper/Resources/map.html`, replace lines 58–78 (the entire
  `getAntisolarPoint` function) with:

  ```javascript
  function getAntisolarPoint(date) {
    const utHours = date.getUTCHours() + date.getUTCMinutes() / 60 + date.getUTCSeconds() / 3600;
    // Longitude where it is solar noon right now (subsolar longitude)
    const subsolarLon = (180 - (utHours - 12) * 15 + 360) % 360 - 180;
    // Solar declination via day-of-year approximation (±23.45°, peaks at summer/winter solstice)
    const dayOfYear = Math.floor((date - new Date(Date.UTC(date.getUTCFullYear(), 0, 0))) / 86400000);
    const declination = 23.45 * Math.sin((2 * Math.PI / 365) * (dayOfYear - 81));
    // Antisolar point is the antipode of the subsolar point
    const antiLon = ((subsolarLon + 180) + 540) % 360 - 180;
    const antiLat = -declination;
    return [antiLon, antiLat];
  }
  ```

  Note: `sin` formula with `dayOfYear - 81` matches summer solstice at day ~172.

### Task 2: Fix WKWebView under App Sandbox by inlining scripts in Swift

**Files:**
- Modify: `WorldClockWallpaper/MapViewController.swift` — `loadMap()` method
- Modify: `WorldClockWallpaper/WorldClockWallpaper.entitlements` — re-enable sandbox

**Context:** `loadFileURL(_:allowingReadAccessTo:)` fails under App Sandbox because the
WKWebView's WebContent process has its own stricter sandbox and cannot read files via `file://`
URLs even with `allowingReadAccessTo`. The solution: read all resources (HTML + JS files) in
Swift (which has normal sandbox access to its own bundle), replace `<script src="...">` tags
with inline `<script>` blocks, and call `loadHTMLString(_:baseURL:)`. The WebContent process
receives self-contained HTML with no external file requests.

- [ ] **Step 1: Re-enable App Sandbox in entitlements**

  Edit `WorldClockWallpaper/WorldClockWallpaper.entitlements`, change:
  ```xml
  <key>com.apple.security.app-sandbox</key>
  <false/>
  ```
  to:
  ```xml

  <true/>
  ```

- [ ] **Step 2: Rewrite `loadMap()` in MapViewController.swift to inline scripts**

  Replace the current `loadMap()` implementation (lines 40–47) with:

  ```swift
  private func loadMap() {
      guard let htmlURL = Bundle.main.url(forResource: "map", withExtension: "html"),
            var html = try? String(contentsOf: htmlURL, encoding: .utf8) else {
          NSLog("WCW: map.html not found in bundle")
          return
      }

      // Inline every <script src="filename.js"></script> from the bundle.
      // This is required under App Sandbox: WKWebView's WebContent process cannot
      // read file:// URLs from the app bundle, but Swift can.
      let scriptPattern = try! NSRegularExpression(
          pattern: #"<script src="([^"]+\.js)"></script>"#
      )
      let range = NSRange(html.startIndex..., in: html)
      let matches = scriptPattern.matches(in: html, range: range)

      // Process in reverse order so string offsets stay valid
      for match in matches.reversed() {
          guard let fileNameRange = Range(match.range(at: 1), in: html),
                let fileURL = Bundle.main.url(
                    forResource: String(html[fileNameRange].dropLast(3)),
                    withExtension: "js"
                ),
                let src = try? String(contentsOf: fileURL, encoding: .utf8),
                let fullMatchRange = Range(match.range(at: 0), in: html) else { continue }
          html.replaceSubrange(fullMatchRange, with: "<script>\(src)</script>")
      }

      webView.loadHTMLString(html, baseURL: Bundle.main.resourceURL)
  }
  ```

  Note: `dropLast(3)` strips `.js` from the filename to use with `url(forResource:withExtension:)`.

## Debugging Reference

**White screen with sandbox ON:**
- The WebContent process cannot read any external resource. After inlining scripts, the only
  remaining external call would be if JS does `fetch()` or `new XMLHttpRequest()`. After Task 1,
  `suncalc.js` is gone; `world-110m.json` is injected by Swift; there are no other fetches.
- Check Console.app → filter "WorldClockWallpaper" for "WebKit" or "sandbox" messages.

**Wrong terminator shape (horizontal band instead of ellipse):**
- Verify `dayOfYear` calculation: should be 1 on Jan 1, ~172 on summer solstice (June 21).
- At equinox (day 81), declination should be 0° → antisolar point at [antiLon, 0°].
- At summer solstice (day 172), declination ≈ +23.45° → antiLat ≈ −23.45° (Tropic of Capricorn).

**Build error after deleting suncalc.js:**
- Ensure the `<script src="suncalc.js">` line was removed from map.html in Task 1 Step 2.
- Check `project.yml` — suncalc.js is included via `path: WorldClockWallpaper/Resources` glob,
  deletion is sufficient (no explicit entry to remove).
